# 共享号封号管理系统 — 产品文档

## 一、产品概述

共享号封号管理系统用于识别和处理平台上存在的共享账号行为。系统包含两个角色端：

- **作者端**：作者通过作品阅读时长榜单发现异常账号，提交举报
- **运营端**：运营人员受理举报，查看详细数据，执行封号/撤销操作，并将处理结果回馈给作者

---

|------|------|------|------|

## 四、作者端功能

### 4.1 举报共享号页

**路由**：`/author/works/[workId]?platform=橙光`

**功能**：作者从橙光后台进入，查看某部作品的用户阅读时长排行榜，从中发现疑似共享号并发起举报。

**入口**：
- 位于橙光后台「作品数据 → 阅读时长榜单」页面
- 在「导出excel」按钮右侧增加「举报共享号」按钮
- 点击后**新窗口打开**本页面，自动携带当前已选的作品 ID 和平台参数
- URL 格式：`/author/works/{workId}?platform={平台名}`

**页面元素**：

- **顶部信息栏**：
  - 页面标题「举报共享号」
  - 只读展示当前作品名称 + 平台（Tag 形式，如「电竞之柠檬驾到 | 橙光」）
  - 作品和平台由 URL 参数确定，页面内不可切换

- **筛选区**：
  - 时间范围（分段选择器：全部 / 按月）
    - **默认选中「按月」**，自动定位最新可用月份
    - 选择「按月」时，右侧显示月份选择器，可切换具体年月
    - 选择「全部」时，月份选择器隐藏，查询全部历史数据
    - **数据更新规则**：每月 1 日更新上个月数据（如：3 月 1 日起可查看 2 月数据）
  - 查询按钮（切换时间范围后刷新榜单数据）

- **榜单表格**：
  | 列名 | 位置 | 说明 |
  |------|------|------|
  | 勾选框 | 最左侧，固定 | 用于批量选择用户 |
  | 用户昵称 | - | 用户显示名，可点击 |
  | 用户 UID | - | 用户唯一标识 |
  | 阅读时长 | - | 该作品的阅读小时数（支持排序） |
  | 移动端阅读时长 | - | 移动端设备的阅读小时数 |
  | PC 阅读时长 | - | PC 端的阅读小时数 |
  | 用户排名 | - | 按阅读时长自动排序 |
  | 操作 | 最右侧，固定 sticky | 整合举报入口与状态展示（见下方说明） |

  **操作列交互规则**：
  - 未举报 → 显示橙色「举报」按钮，点击弹出举报弹窗
  - 已举报 → 显示状态文字（待处理 / 已封号 / 已驳回）+ 蓝色「查看进度」链接，跳转到我的举报记录
  - 不单独设置「举报状态」列，状态信息整合到操作列中，保持表格简洁

- **批量操作**：
  - 左侧勾选框支持多选
  - 选中后顶部浮出操作条：已选 N 人、可举报 N 人、「批量举报」按钮、「取消选择」
  - 已举报的用户自动跳过，不重复提交

- **右上角**：「我的举报记录」链接，跳转到 `/author/reports`

### 4.2 举报弹窗

**触发**：点击单条「举报」按钮或「批量举报」按钮

**弹窗内容**：

1. **被举报用户摘要**
   - 单个用户：显示昵称、UID、阅读时长
   - 批量用户：显示「已选择 N 个用户」+ 用户标签列表

2. **举报原因**（多选标签，至少选一个）
   - 阅读时长异常偏高
   - 疑似多人共用账号
   - 设备数量异常
   - 付费与使用不匹配
   - 其他

3. **补充说明**（文本框，选填，限500字）

4. **提交**：调用 API 创建举报记录，成功后榜单中该用户状态自动刷新为「待处理」

**去重规则**：同一作品 + 同一用户 + 同一作者，不允许重复举报。

### 4.3 我的举报记录页

**路由**：`/author/reports`

**功能**：作者查看自己提交的所有举报记录及处理进度。

**表格列**：

| 列名 | 说明 |
|------|------|
| 举报时间 | 提交举报的时间（支持排序） |
| 作品名称 | 被举报作品 |
| 被举报用户 | 昵称 + UID |
| 阅读时长 | 该用户在该作品的阅读小时数 |
| 举报原因 | 以标签形式展示所选原因 |
| 处理状态 | 待处理（蓝）/ 处理中（橙）/ 已封号（绿）/ 已驳回（红） |
| 处理结果 | 运营填写的处理说明 |
| 处理时间 | 运营处理的时间 |

**展开行详情**：点击行可展开查看完整信息，包括作品ID、补充说明、封号天数等。

**状态筛选**：支持按处理状态列筛选。

## 五、运营管理端功能

### 5.1 管理首页（举报记录列表）

**路由**：`/`

**功能**：运营人员查看和处理所有共享号举报记录。

**页面元素**：

- **统计卡片**（顶部4个）：
  | 指标 | 说明 |
  |------|------|
  | 当前记录数 | 筛选后的总条数 |
  | 待处理 | 状态为 pending 的数量 |
  | 已处理 | 状态为 processed 的数量 |
  | 总举报人数 | 所有记录的举报人总数 |

- **筛选区**：
  - 关键词搜索（支持 UID / 昵称 / 作品 ID / 作品名）
  - 状态筛选（全部 / 待处理 / 已处理）
  - 查询 / 重置按钮
  - **导出 Excel** 按钮（导出当前筛选结果为 `.xlsx` 文件）

- **批量操作条**（选中记录后出现）：
  - 已选数量显示
  - 批量处理按钮
  - 批量撤销按钮
  - 取消选择

- **主表格**：

  | 列名 | 数据维度 | 说明 |
  |------|----------|------|
  | 用户 UID | 用户 | 可点击跳转到用户详情页 |
  | 用户昵称 | 用户 | - |
  | 作品 ID | 作品 | - |
  | 作品名称 | 作品 | - |
  | 举报人 | 作品 | 显示人数，点击弹出 Popover 查看详情 |
  | 作品历史阅读 | **作品** | 该用户在该作品的历史阅读时长 |
  | 作品当月阅读 | **作品** | 该用户在该作品的当月阅读时长 |
  | 用户历史付费 | **用户** | 该用户在全平台的历史付费总额 |
  | 用户近月付费 | **用户** | 该用户在全平台的近一个月付费 |
  | 当月设备数 | 用户 | 点击弹出 Popover 查看设备明细 |
  | 状态 | - | 待处理（蓝）/ 已处理（绿） |
  | 封号时长 | - | 显示天数 + 第几次封号 |
  | 更新时间 | - | - |
  | 操作 | - | 处理 / 撤销按钮 |

  - 支持按数值列排序（阅读时长、付费金额、设备数）
  - 支持多选（勾选框）
  - 点击行打开详情抽屉

### 5.2 操作确认弹窗

**触发**：点击单条或批量的「处理」/「撤销」按钮

**弹窗内容**：
- 操作类型提示（处理/撤销）
- 影响记录数量
- 备注输入框：**处理操作备注必填**，撤销操作备注选填
- 确认/取消按钮

### 5.3 详情抽屉

**触发**：点击表格中任意一行

**抽屉内容**：

1. **基本信息**（Descriptions 表格）：
   - 用户 UID、昵称
   - 作品 ID、名称
   - 作品历史/当月阅读时长
   - 用户历史/近月付费
   - 当月设备数
   - 状态

2. **封号信息**（仅已处理记录显示）：
   - 封号次数（第 N 次）
   - 封号时长（N 天）
   - 封号到期时间
   - 备注、操作人、处理时间

3. **举报人列表**：展示所有举报人的姓名、举报原因、举报时间

4. **登录设备明细**（表格）：
   | 列名 | 说明 |
   |------|------|
   | 设备名称 | 带设备类型图标 |
   | 设备类型 | 手机（蓝）/ 平板（橙）/ PC（绿） |
   | 最后活跃时间 | - |

5. **操作日志**（时间线）：
   - 记录每次处理/撤销操作
   - 显示操作类型、操作人、备注、时间

### 5.4 用户详情页

**路由**：`/user/[uid]`

**功能**：聚合查看某个用户的全部被举报记录。

**页面元素**：

- **用户信息卡片**：
  - 用户 UID（可复制）
  - 用户昵称
  - 被举报作品数
  - 总举报人数

- **统计卡片**（4个）：
  | 指标 | 说明 |
  |------|------|
  | 待处理 | 该用户待处理的举报数 |
  | 已处理 | 该用户已处理的举报数 |
  | 累计封号次数 | 历史封号总次数 |
  | 最高当月设备数 | 所有记录中最大的设备数 |

- **被举报记录表格**：
  - 列包含：作品 ID、作品名、举报人 Popover、阅读时长、付费、设备数 Popover、状态、封号时长、更新时间、操作
  - 点击行打开详情抽屉
  - 支持处理/撤销操作

### 5.5 导出 Excel

**触发**：管理首页点击「导出 Excel」按钮

**导出内容**：当前筛选结果的全部字段，文件名格式 `封号记录_YYYY-MM-DD.xlsx`

## 六、核心业务规则

### 6.1 封号时长递增规则

| 封号次数 | 封号天数 | 说明 |
|----------|----------|------|
| 第 1 次 | 3 天 | 首次封号 |
| 第 2 次 | 7 天 | - |
| 第 3 次 | 14 天 | 第 2 次起每次递增 7 天 |
| 第 4 次 | 21 天 | - |
| 第 N 次（N≥2） | N × 7 天 | 公式：`banCount * 7` |

- 封号次数按该用户历史上所有被处理（processed）的记录总数统计
- 处理时自动计算封号天数和到期时间
- 撤销操作会清除封号信息（天数、到期时间）

### 6.2 操作日志

- 每次处理或撤销操作自动记录日志
- 日志包含：操作类型、操作人、备注、时间
- 在详情抽屉中以时间线形式展示

### 6.3 举报去重

- 作者端：同一作品 + 同一用户 + 同一作者，不允许重复举报
- 批量举报时自动跳过已举报用户

### 6.4 数据维度说明

| 字段 | 维度 | 含义 |
|------|------|------|
| 作品历史阅读 | 作品 | 用户在该作品上的全部阅读时长 |
| 作品当月阅读 | 作品 | 用户在该作品上的当月阅读时长 |
| 用户历史付费 | 用户 | 用户在全平台的累计付费总额 |
| 用户近月付费 | 用户 | 用户在全平台的近一个月付费 |
| 当月设备数 | 用户 | 用户当月登录设备总数 |

## 七、举报流程

```
作者发现异常 → 在榜单页选中用户 → 点击举报 → 填写原因和说明 → 提交
                                                                    │
                                                                    ▼
                                                        运营管理后台收到举报
                                                                    │
                                                                    ▼
                                                 运营查看详情（阅读数据、设备、付费）
                                                                    │
                                                              ┌─────┴─────┐
                                                              ▼           ▼
                                                         确认封号      驳回举报
                                                         （必填备注）  （选填备注）
                                                              │           │
                                                              ▼           ▼
                                                 自动计算封号天数    状态改为已驳回
                                                 写入操作日志        写入操作日志
                                                              │           │
                                                              ▼           ▼
                                                    作者在「我的举报」页面查看处理结果
```

## 八、API 接口清单

### 运营端

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/reports` | 查询举报列表（支持 keyword、status 筛选） |
| POST | `/api/reports/[id]/process` | 单条处理（body: remark, operator） |
| POST | `/api/reports/[id]/revoke` | 单条撤销（body: remark, operator） |
| POST | `/api/reports/batch` | 批量处理/撤销（body: ids, action, remark, operator） |
| GET | `/api/reports/[id]/logs` | 查询某条记录的操作日志 |
| GET | `/api/users/[uid]/reports` | 查询某用户的全部举报记录 |

### 作者端

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/author/works/[workId]/leaderboard` | 获取作品阅读时长榜单 |
| POST | `/api/author/reports` | 提交举报（body: work_id, targets, reasons, description） |
| GET | `/api/author/reports` | 查询我的举报记录 |
